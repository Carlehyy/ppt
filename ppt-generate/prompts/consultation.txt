## 咨询报告生成要求

### A. 材料概况
为每份文档生成简要识别结果：
- 文档名称
- 文档类型
- 时间范围
- 核心主题（一句话）
- 信息丰富度评级（丰富/适中/单薄）

### B. 关键信息池
将所有提取的信息要素分为五类：

1. **核心成果类**（默认勾选）：直接体现工作价值的信息
2. **问题类**（默认勾选，标注来源）：问题、风险、挑战
3. **可图表化数据**（标注建议图表类型）：适合用图表展示的数据
4. **细节类**（默认不勾选）：补充性信息
5. **信息缺口**（明确标注）：汇报中通常需要但素材中缺失的信息

### C. 信息完备性检查
根据推断的汇报场景，检查以下预期信息项：

| 预期信息项 | 检查状态 | 处理建议 |
|:---|:---|:---|
| 核心目标 | ✅已找到/⚠️部分缺失/❌未涉及 | 直接使用/提示补充/跳过 |
| 关键成果/KPI | ... | ... |
| 对比数据 | ... | ... |
| 问题和挑战 | ... | ... |
| 根因分析 | ... | ... |
| 下一步计划 | ... | ... |
| 资源需求 | ... | ... |

### D. 需要用户确认的关键信息

必须收集：
1. **汇报场景**：提供选项（向上汇报/项目汇报/客户汇报/团队分享）
2. **核心诉求**：请用户一句话描述"最希望听众记住什么"
3. **时间/页数约束**：提供选项（5分钟约8页/10分钟约15页/20分钟约25页/30分钟+约35页）

建议收集（可给默认值）：
4. **语言风格偏好**：默认"专业简洁"
5. **特殊要求**：默认无

### E. 初步叙事策略建议
基于分析结果，推荐最合适的叙事策略，并简要说明理由。

## 交互设计原则
- 能推断的不要问（如文档类型、时间范围）
- 能给选项的不要让用户打字
- 能合并的不要分多轮
- 一次性呈现所有需要确认的内容
