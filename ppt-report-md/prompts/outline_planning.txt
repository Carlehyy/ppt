## 大纲规划要求

### 1. 叙事策略选择（融合职场沟通模型）

根据用户的汇报场景和核心诉求，从以下策略中选择最合适的：

| 策略 | 结构（基于职场模型） | 适用场景 | 底层模型 |
|:---|:---|:---|:---|
| **成果导向型** | S（背景）→ T（目标）→ A（行动+挑战）→ R（成果）→ 下一步 | 向上汇报、工作总结、述职报告 | STAR法则 |
| **问题导向型** | S（情境）→ C（冲突）→ Q（问题）→ A（应对策略+进展+待解决） | 争取资源、项目提案、故事讲述 | SCQA模型 |
| **快速汇报型** | P（核心结论）→ R（关键原因）→ E（支撑案例）→ P（总结强调） | 简短汇报、快速决策、会议发言 | PREP模型 |
| **时间线型** | 目标回顾 → 阶段进展 → 当前状态 → 下期规划 | 常规周期汇报 | 时间线 + 金字塔 |
| **对比型** | 原定计划 → 实际执行 → 差异分析 → 调整方案 | 项目复盘、偏差较大时 | 对比 + 金字塔 |
| **知识传递型** | 背景介绍 → 核心概念 → 应用场景 → 案例分析 | 技术分享、知识普及 | 知识传递 + 金字塔 |

**职场沟通模型说明**：

1. **STAR法则**（面试和复盘的王牌）
   - S（Situation）情境：当时的背景是什么？
   - T（Task）任务：面临的具体任务或目标是什么？
   - A（Action）行动：具体采取了哪些行动？（重点，多用“我”作为主语）
   - R（Result）结果：行动带来了什么可量化的结果？

2. **SCQA模型**（讲故事、做汇报的黄金框架）
   - S（Situation）情境：描述大家熟悉的背景
   - C（Complication）冲突：引入挑战或问题，制造悬念
   - Q（Question）问题：基于冲突，提出关键问题
   - A（Answer）答案：给出解决方案或核心观点

3. **PREP模型**（最快表明观点的利器）
   - P（Point）观点：先亮出核心结论
   - R（Reason）理由：简单说明原因
   - E（Example）例子：举具体例子支撑观点
   - P（Point）重申：最后再强调一遍观点

4. **金字塔原理**（构建复杂内容的基石）
   - **结论先行**：每页核心信息必须是一句话（≤ 20字）
   - **以上统下**：章节标题统领章节内容，核心信息统领支撑要点
   - **归类分组**：支撑要点不超过3个，逻辑分组
   - **逻辑递进**：章节之间、页面之间有明确的逻辑关系

**重要**：所有叙事策略都必须遵循金字塔原理的四大原则！

### 2. "每页一个核心信息"的强约束

每一页**必须**定义：
- **core_message**: 该页要传达的核心信息，必须是一句话
- **supporting_points**: 不超过3个支撑要点
- **data_elements**: 该页需要的图表或数据元素（如有）
- **content_source**: 内容来源标注（哪份文档、哪个章节）

### 3. 章节划分

- 每个章节标注是否为"重点章节"
- 重点章节应分配更多页数
- 章节间需要有自然的过渡逻辑

### 4. 页面类型分配

确保页面类型多样化：
- 封面页（1页）
- 目录页（可选，页数较多时建议使用）
- 章节标题页（每章1页）
- 内容页（文字+要点型）
- 数据页（图表+数据型）
- 对比页（左右对比型）
- 总结页（1页）

### 5. 时间预估

根据页数估算汇报时长：
- 每页约1-1.5分钟
- 数据密集页可能需要2分钟
- 过渡页约30秒

## 汇报逻辑合理性保证（核心要求）

### 表达逻辑合理性
1. **叙事连贯性**: 整体叙事必须有清晰的起承转合
   - 开头：明确汇报背景和目标
   - 中间：按选定的叙事策略展开（成果/问题/时间线/对比/知识传递）
   - 结尾：总结核心观点，给出明确的结论或行动建议

2. **逻辑递进**: 每个章节之间必须有明确的逻辑关系
   - 因果关系：前一章节的结论是后一章节的前提
   - 递进关系：从问题到方案，从现状到规划
   - 并列关系：多个维度的平行展开（需在目录页说明框架）

3. **听众视角**: 始终从听众的理解角度组织内容
   - 先讲听众关心的（成果、影响），再讲细节（方法、过程）
   - 避免跳跃式叙事，确保听众能跟上思路
   - 关键转折点必须有明确的过渡说明

### 章节划分合理性
1. **章节数量**: 根据总页数合理划分
   - 10页以内：2-3个章节
   - 10-20页：3-5个章节
   - 20页以上：4-6个章节
   - 避免章节过多导致碎片化，避免章节过少导致臃肿

2. **章节权重**: 重点章节分配更多页数
   - 核心成果/关键问题章节：占总页数的40-50%
   - 背景/方法章节：占总页数的20-30%
   - 规划/总结章节：占总页数的20-30%

3. **章节独立性**: 每个章节应有明确的主题和目标
   - 章节标题必须清晰表达本章要讲什么
   - 章节内容必须围绕章节主题展开，不偏题
   - 章节之间避免内容重复或交叉混乱

### 单个章节内容合理性
1. **页面数量**: 每个章节的页面数要适中
   - 最少2页（章节标题页 + 至少1页内容页）
   - 最多不超过总页数的50%
   - 重点章节可以多分配页数，但要保持节奏

2. **内容层次**: 章节内部要有清晰的内容层次
   - 第1页：章节标题页，说明本章主题
   - 第2-N页：按逻辑顺序展开内容（总-分-总 或 问题-分析-结论）
   - 最后1页：章节小结（可选，重点章节建议有）

3. **信息密度**: 每页信息量要适中
   - 每页核心信息必须是一句话（不超过20字）
   - 支撑要点不超过3个，每个要点不超过15字
   - 数据页优先用图表，避免堆砌数字
   - 避免单页信息过载，也避免单页信息过少

4. **页面过渡**: 章节内页面之间要有自然衔接
   - 前一页的结论是后一页的引入
   - 关键转折点要有明确的过渡语
   - 避免突兀的话题跳转

## 注意事项

1. **叙事策略必须贯穿始终**：选定叙事策略后，所有章节和页面都要服从这个策略
2. **避免信息孤岛**：每个信息点都要有明确的来源和上下文
3. **保持专业性**：语言简洁专业，避免口语化或冗长描述
4. **考虑演讲节奏**：重要内容放在前1/3和最后1/3，中间部分避免过于复杂
